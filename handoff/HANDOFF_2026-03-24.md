# Handoff Técnico

## Estado General

El sitio legado sigue en [`index.html`](../index.html) y **no fue reemplazado**.

Se montó una capa nueva, aislada y compilable en [`app/`](../app), basada en:

- `React + Vite`
- contrato de datos externo en [`data/2024.json`](../data/2024.json)
- validador de consistencia en [`scripts/validate.js`](../scripts/validate.js)

El shell React compila en `dist/app` con:

```bash
npm run app:build
```

## Fuente de Verdad

La fuente de verdad del anuario migrado es:

- [`data/2024.json`](../data/2024.json)

El shell solo consume `data/*.json`.

## Advertencias Actuales del Dataset 2024

El validador hoy deja 4 `WARN` reales:

1. `financiera`
   `ejecutadoCentros + ejecutadoProyectos != totalEjecutado`
2. `productividad.congresos`
   el desglose suma `6842`, no `6867`
3. `capitalHumano`
   la pirámide suma `1717` hombres, no `1737`
4. `territorio`
   el detalle regional suma `475`, no `494`

No se corrigieron automáticamente porque el objetivo fue **hacer visibles las discrepancias**, no inventar datos.

## Scaffold para 2025

Se agregó:

- [`scripts/scaffold-year.js`](../scripts/scaffold-year.js)

Y se puede generar un año nuevo así:

```bash
npm run scaffold:year -- 2025
```

Eso crea por defecto:

- `handoff/scaffolds/2025.scaffold.json`

Ese scaffold:

- conserva la forma del contrato
- vacía los datos sensibles al año
- no entra al shell automáticamente
- no rompe el validador porque queda fuera de `data/`

## Siguiente Paso Recomendado

El siguiente frente natural ya no es refactor. Es uno de estos dos:

1. Poblar `2025` con el mismo contrato y encender el selector multi-año real.
2. Construir vistas comparativas `2024 vs 2025` sobre el shell ya existente.

Orden recomendado:

1. generar scaffold 2025
2. poblar `data/2025.json`
3. validar con `npm run validate:data`
4. mover el scaffold completado a `data/2025.json`
5. agregar comparativas y transiciones entre años

## Nota Importante para Otro Modelo

No asumir que el dato del 2024 está reconciliado solo porque el shell compila. El shell hoy renderiza bien, pero también expone inconsistencias reales del dataset. Antes de publicar o replicar lógica de cálculo, revisar:
